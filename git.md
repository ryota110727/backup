# Gitとは
バージョン管理システム

どのファイルを誰が、いつ、どのように、なんのために変更したかを残せる

元の状態にも戻せる

## gitの仕組み
* 変更履歴を順々に記録する
* 記録する際はメッセージをつける

* リポジトリにファイルとファイルの変更履歴を保存している
* commit : 個人リポジトリに変更履歴を記録する
* push : 共有リポジトリに変更を共有

### 複数人の開発の場合
* pull : 共有リポジトリの情報を同期
*
